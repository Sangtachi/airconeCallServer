서버 ENV (Nest):
  SUPABASE_URL=
  SUPABASE_SERVICE_ROLE_KEY=   서버 전용
  SUPABASE_STORAGE_ORDER_PHOTOS_BUCKET=order-photos   (선택; 기본 order-photos)
  x-admin-role / x-technician-id 는 이번 단계의 임시 헤더입니다. 보안 인증은 다음 스프린트에서 Supabase Auth 또는 서버 세션으로 교체합니다.
  EMERGENCY_DEFAULT_PRODUCT_ID=  (선택 — 긴급 리드 자동 주문 초안의 기본 설치 상품 UUID; 미설정 시 첫 활성 설치 상품 폴백)
  ORDER_PHOTOS_SIGNED_URL_TTL_SEC=900  (0 이면 목록에 DB url 그대로 — 공개 버킷용)

Storage(작업 사진 presigned / 확인 URL):
  - 대시보드 Storage 에 버킷 생성(이름은 ENV 와 동일 또는 기본 order-photos).
  - 업로드: 서비스 롤 또는 signed upload 정책으로 presigned PUT 허용.
  - 비공개 버킷: Supabase 에서 6) DDL 로 order_photos.storage_bucket/object_path 저장 후
    Nest ENV `ORDER_PHOTOS_SIGNED_URL_TTL_SEC`(초, 예: 900) 를 설정하면 목록 API 가 signed URL 을 내려줍니다.

정산 감사(settlement_events): 6) 적용 후 PATCH/confirm/soft-delete 시 idempotency-key + 액터 기록(멱등키 중복은 스킵).

RLS 정책은 프로젝트에 맞게 별도 설계하면 됩니다(서버는 서비스 롤로 우회).
